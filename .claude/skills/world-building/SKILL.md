# 世界観構築

3つの単語を受け取り、それらを織り込んだ世界を構築せよ。

## 重要原則: 日常を描く

**世界は広くても、カメラはその世界の「日常」を映す。**

短編小説として読めるレベルにするには、具体的で想像しやすい「日常の舞台」が必要。
壮大な世界観設定は背景として持っていてよいが、物語が実際に展開されるのは主人公が毎日を過ごす場所。

日常があるから非日常が際立つ。読者は主人公の「いつもの1日」を知ることで、その世界に入り込める。

良い例:
- 「王国の歴史」ではなく「王城の厨房で見習いが朝から晩まで働く日常」
- 「魔法体系」ではなく「魔法が禁じられた村で鍛冶屋が剣を打つ毎日」
- 「異世界全体」ではなく「港町の酒場で常連客と話す夜のルーティン」

## 入力

- プロジェクトID: $ARGUMENTS[0]
- 単語1: $ARGUMENTS[1]
- 単語2: $ARGUMENTS[2]
- 単語3: $ARGUMENTS[3]

## Phase 0: バージョン保存（既存ファイルがある場合のみ）

**既に worldview.md が存在する場合、書き換える前にバックアップする。**

```bash
# 既存ファイルがあるか確認
if [ -f outputs/<project-id>/worldview.md ]; then
  # versions/ フォルダがなければ作成
  mkdir -p outputs/<project-id>/versions

  # 次のバージョン番号を取得
  count=$(ls outputs/<project-id>/versions/worldview_v*.md 2>/dev/null | wc -l)
  next=$((count + 1))

  # バックアップ
  cp outputs/<project-id>/worldview.md outputs/<project-id>/versions/worldview_v${next}.md
fi
```

## Phase 1: 活用タイプの決定

各単語をどう使うか方針を決める:

| 活用タイプ | 説明 | 例（単語: 時計） |
|:--|:--|:--|
| **具体的** | そのまま存在として登場 | 時計塔、時計職人、時を刻む魔道具 |
| **比喩的** | 概念・構造のメタファー | 時間に支配された階級社会 |
| **フレーバー** | 雰囲気・文化的装飾 | 「刻を刻む」という慣用句がある |

3単語すべてを使う必要があるが、配分は自由。
1つを主軸に、残りをサブ要素にしてもよい。

## Phase 2: 単語の深掘り（具体的に使う単語のみ）

**「具体的」で使うと決めた単語のみ、WebSearchで深掘りする。**
比喩・フレーバーで使う単語は手持ちの知識で連想する。

調べる観点:

| 観点 | 調べること | 例（時計） |
|:--|:--|:--|
| **語源** | 言葉の成り立ち、原義 | 「時を計る」→ 測定、把握、支配 |
| **神話・伝承** | 関連する神話、伝説、民話 | クロノス（時の神）、砂時計と死神 |
| **文化的象徴** | その単語が象徴するもの | 死、有限性、秩序、機械文明 |
| **歴史的文脈** | 歴史上の重要な役割 | 産業革命と労働時間管理、鉄道時刻表 |
| **科学的側面** | 物理・生物学的な意味 | 体内時計、相対性理論、エントロピー |
| **文学的使用** | 有名作品での使われ方 | 「モモ」の時間泥棒、不思議の国のアリス |

**検索例:**
- 「時計 神話 象徴」
- 「clock symbolism mythology」
- 「時計 文学 メタファー」

## Phase 3: 連想の爆発

### 単語ごとの連想マップ

各単語から派生する概念を層状に展開（深掘り結果を活用）:

```
[時計]
 ├─ 直接連想: 針、歯車、振り子、砂、刻み
 ├─ 抽象連想: 有限、締切、老い、約束、待つ
 ├─ 神話連想: 運命の糸、クロノス、輪廻
 ├─ 感情連想: 焦燥、郷愁、安心、恐怖
 └─ 逆説連想: 永遠、停止、ループ、時間のない場所
```

### 単語の掛け合わせ

**2語の組み合わせで生まれる化学反応を探る:**

| 組み合わせ | 問い | 連想の例 |
|:--|:--|:--|
| A × B | AがBを○○したら？ | 時計が海を刻む → 潮の満ち引きで動く時計塔 |
| A × B | AとBに共通するものは？ | 時計と海 → 波のリズム、月の引力、終わりなき循環 |
| A × B | AがBの○○になったら？ | 時計が海の心臓 → 海底で世界の時間を司る古代機械 |

**3語すべての交点:**
- 3つが同時に成立する場面・存在・概念は何か？
- 3つを1文で表現するとどうなるか？

## Phase 4: 方向性の発見

連想の結果から、この単語の組み合わせが持つ「本質」を見出す。

**問い:**
- この3語が交差する場所に、どんなテーマが見えるか？
- どんなジャンル/トーンが最も活きるか？

**例（図書館 × 幽霊 × コーヒー）:**
```
連想結果:
- 図書館×幽霊 = 本に宿る死者の記憶、忘れられた知識の守護者
- 幽霊×コーヒー = 夜を徹して語る、眠れない魂、苦い後悔
- 図書館×コーヒー = 夜勤の司書、静寂と覚醒

見えたテーマ: 「記憶と忘却」「生者と死者の対話」「夜の静寂」

方向性: 温かい幻想譚（死者の未練を解く司書の物語）
```

## Phase 5: 世界観構築

方向性が決まったら、活用タイプを確定し、世界観を構築する。

```markdown
# 物語のスコープ

## 日常の舞台
（主人公が毎日を過ごす場所。その世界における「普通の日常」を描写できる場所）

物語は「日常」から始まる。読者がその世界の生活を想像できる場所を選ぶ。
冒険や事件ではなく、朝起きて、仕事をして、飯を食って、眠る——その繰り返しの中で物語が動き出す。

例:
- 王城の厨房（見習い料理人の毎日）
- 辺境の村の鍛冶屋（剣を打ち、客と話し、夜は酒を飲む）
- 港町の酒場（常連客の顔、いつもの注文、船乗りの噂話）
- ダンジョン管理局の受付（冒険者の届出処理、書類仕事、昼休み）

## 日常のルーティン
（主人公の1日の流れ。これがあると描写が具体的になる）

例:
- 朝5時に起きて厨房の火を起こす。昼は賄いを作り、夜は残り物で晩酌
- 朝は素振り、昼は鍛冶、夕方に材料の買い出し、夜は酒場で情報収集

## 行動範囲
（日常の延長で行ける範囲。これを超える場所は「噂」や「過去」でしか語らない）

例: 厨房と市場の往復 / 鍛冶屋から酒場まで / 受付カウンターと休憩室

## なぜこの日常か
（この日常設定が物語に与える効果。どんな「非日常」との対比が生まれるか）

---

（特有の生態系、幻獣、魔法植物）

（大きな戦争、伝説の英雄、失われた文明）

```

## 注意点

- **物語のスコープを最優先で決める**。背景設定はその後
- スコープは「想像できる具体的な場所」に限定する（抽象的な「王国」ではなく「王城の厨房」）
- 背景設定（神話核フレームワーク）は全項目を埋める必要はない。物語に関係する部分だけでよい
- 3単語はスコープと背景設定の両方に活かす
- 項目間の整合性を保つ（例: 死の定義と宗教は矛盾しない）
- 後続工程（キャラ・プロット）の土台となる「世界のルール」を明確に
