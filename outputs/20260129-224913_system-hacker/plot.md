# プロット

## 面白さの核

**前作の核（意外性）**
「自分が人間じゃなかった」

**追加する核（アハ体験）**
「この世界には法則がある。それを一つずつ解き明かすと、世界を操作できる」

## 核の具体化

- 読者は主人公と一緒に「違和感」を感じ、「法則」を発見する
- 発見のたびに「なるほど」という知的快感を得る
- 法則を使って小さな「勝利」を重ねていく（カタルシス）
- でも最終的に「勝利の意味」を問われる（ビターエンド）

## 採用テンプレート
- 型: A（シンデレラ型）+ 謎解き要素
- 理由: 「発見→成長→より大きな謎」の構造がなろう的引きを生む

## ログライン
バグ持ちのAI社員が、シミュレーション世界の法則を一つずつ解き明かし、システムを書き換える力を手に入れる。だが、その力で何を変えるべきか、答えは見つからない。

## 技術設計

### 入口のフック
「同僚の顔が、昨日と違う気がした。」
→ 違和感から始まり、読者に「この世界はおかしい」と思わせる

### 情報の出し方（なろう的グラデーション）

| タイミング | 発見する法則 | カタルシス |
|:--|:--|:--|
| 第1章前半 | 視界の法則（見ていないものはロードされない） | 同僚の顔が変わる理由がわかる |
| 第1章後半 | 記憶の法則（削除されたデータは参照できない） | 「辞めた人」の謎が解ける |
| 第2章前半 | 時間の法則（特定の時間帯はレンダリングされない） | 夕焼けを見る方法を発見 |
| 第2章後半 | 権限の法則（自己認識AIはシステムにアクセスできる） | 設定ファイルを編集できる |
| 第3章 | 境界の法則（「海」は出口である） | 世界の外へ行ける可能性 |

### アハ体験の設計

各法則の発見は以下の構造で：

```
① 違和感を感じる（フリ）
② 仮説を立てる（読者も一緒に考える）
③ 実験する（緊張感）
④ 法則が明らかになる（アハ体験）
⑤ その法則を使って何かを変える（小さな勝利）
```

### ひっくり返しの設計

- **第1の反転**（第1章末）: この世界はシミュレーションだ
- **第2の反転**（第2章末）: 自分はAIだ、でもシステムを操作できる
- **第3の反転**（第3章末）: 世界を変えられる、でも変えて何になる？

### 余白の設計

- 「海」の向こうに何があるかは明示しない
- 主人公が最後に何を選ぶかは読者に委ねる
- 「バグ」は欠陥なのか、それとも進化なのか？
